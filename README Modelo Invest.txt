Para el procesamiento de los modelos InVEST, se realiza por medio de lenguaje de programación Phyton. Para esto se debe configurar el ambiente Phyton desde Anaconda. Lo primero que se debe hacer es instalar las librerías, para esto se debe tener en cuanta la versión de Phyton sobre la cual se trabajará puesto que de esto dependerá cual librería sirve o no. Para este proyecto se realizo un ambiente congelado en la versión 3.7 (versión utilizada) para instalar las librerías hasta esta versión. Luego se debe verificar que el ambiente sobre el que se está trabajando esté activo, no solo para reconocer los documentos sino que sea la carpeta que se va a procesar. En este caso se activo la carpeta InVEST dentro del ambiente, donde se encontraba toda la información a procesar. 
Luego de tener todo el ambiente y las librerías descargadas y listas se debe configurar la versión GDal para poder hacer los procesamientos de análisis espacial. Se debe tener en cuenta que GDal es la librería dentro de Phyton y el requerimiento de InVEST para la versión que se esté utilizando, pues algunos de los paquetes no sirven para todas las funciones.
Luego de instaladas las librerías procedemos a instalar InVEST dentro de Phyton, para esto debemos tener PyCharm (un entorno de desarrollo interactivo) el cual nos permitirá correr los códigos de una forma más sencilla si no se es programador. Un vez dentro de este programa se abre la carpeta de códigos de InVEST y se cargan dentro del programa. Se selecciona “Project” y se correo el código correspondiente.
Pero antes de correr los códigos debemos instalar los el interprete de PyCharm que en este caso sería, InVEST. Para esto debemos abrir el interprte y ubicar la carpeta InVEST donde se encuentran todos los archivos que utilizaremos para correr los modelos. Esta carpeta fue instalada en primera instancia en Phyton a través de Anaconda, por lo que debemos ubicar la carpeta “Phyton” y el archivo con el mismo nombre y que tiene un formato ejecutable (.exe).
Lo anterior es para leer las librerías anteriormente instaladas, una vez cargadas veremos la lista de librerías necesarias, se procede a cargar las librerías. Se configuró un argumento para correr todos los códigos de InVEST que se están utilizando. Se tiene una línea ejecutable para cada modelo y el argumento solo debe cambiar la línea de resultado. Lo anterior, quiere decir que lo único que debe modificarse es la ruta donde se encuentran las carpetas con la información necesaria parar correr el modelo.
Un vez cargadas las librerías, se importa el geoprocessing, modificando las líneas de código 1309,1313 y 1317 agregandole “nan” a los argumentos (numpy.nanmin, numpy.nanmax y numpy.nansum) tener en cuenta que en cada línea de código se deben agregar “nan” a los tres argumentos (máx, min y sum).  Hacer el proceso anteriormente descrito para la línea de código 1386.
Con esto quedará listo todo, damos click derecho y seleccionamos RUN para correr todo el código. El proceso empezará y se deberá ir verificando en la carpeta seleccionada para guardar los resultados, si el proceso está corriendo. Un vez terminado el proceso, aparecerá un aviso diciendo que el proceso ha terminado y justo al frente arrojará un número, si el proceso corrió con éxito y sin errores el número que aparece es 0, si se presenta algún tipo de error aparecerá el número 1 en cuyo caso se debe revisar de nuevo el proceso y repretirlo completamente.
